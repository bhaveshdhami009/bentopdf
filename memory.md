- **pdf-multi-tool.ts**: The `updatePageDisplay()` function reconciles the entire DOM map for PDF thumbnails, which can destroy lazy loading observers if called unnecessarily. Functions modifying the array slightly should instead use manual DOM modifications combined with `updatePageNumbers()` to maintain high performance and preserve lazy loading states.
- **updatePageNumbers()**: This is a lightweight alternative to `updatePageDisplay()` that just re-indexes elements and reattaches listeners without restructuring the DOM. I've enhanced it to also update selection states and icons.
